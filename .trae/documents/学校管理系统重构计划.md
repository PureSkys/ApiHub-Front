# 学校管理系统全面重构计划

## 一、项目概述

基于后端API文档（openapi.json），对学校管理系统进行完全重构，确保前端与后端API完全对齐，提供现代化、用户友好的管理界面。

## 二、API端点总览

### 2.1 学校管理 API
| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /school/ | 获取学校列表 |
| POST | /school/ | 创建学校（仅超级管理员） |
| GET | /school/{school_id} | 获取单个学校 |
| PUT | /school/{school_id} | 更新学校 |
| DELETE | /school/{school_id} | 删除学校（仅超级管理员） |
| GET | /school/{school_id}/classes | 获取学校下所有班级 |
| GET | /school/{school_id}/exams | 获取学校下所有考试 |

### 2.2 班级管理 API

|------|------|------|

| GET | /school/class/{class_id} | 获取单个班级 |

### 2.6 统计分析 API
| 方法 | 路径 | 功能 |
|------|------|------|

| GET | /school/stats/class/{class_id}/detail | 班级详细统计 |
| GET | /school/stats/class/{class_id}/exam/{exam_id} | 按班级统计某次考试成绩 |

| GET | /school/stats/exam/{exam_id}/subject/{subject} | 按科目统计某次考试 |
| GET | /school/stats/student/{student_id} | 学生成绩趋势分析 |

## 五、实施步骤

### 第一阶段：API层重构
1. 根据API文档重新定义所有TypeScript接口
2. 重写所有API调用函数，确保路径和参数正确
3. 添加完整的错误处理和类型安全

### 第二阶段：基础管理模块
1. **学校管理页面**：列表、创建、编辑、删除
2. **班级管理页面**：列表、创建、批量导入、编辑、删除
3. **学生管理页面**：分页列表、创建、批量导入、编辑、删除、查看成绩

### 第三阶段：考试与成绩模块
1. **考试管理页面**：列表、创建、编辑、删除、查看成绩
2. **成绩管理页面**：分页列表、创建、批量导入、编辑、删除

### 第四阶段：统计分析模块
1. **学校概览统计**：学校维度数据展示
2. **考试综合分析**：考试概览、成绩分布、排名、班级对比
3. **班级成绩分析**：班级各科统计
4. **学生成绩趋势**：学生历次考试成绩变化
5. **单科深度分析**：科目详细统计

### 第五阶段：管理员与日志模块
1. **管理员管理页面**：创建、编辑、删除、启用/禁用
2. **操作日志页面**：日志列表、用户日志查询

### 第六阶段：用户体验优化
1. 筛选器状态持久化
2. 搜索功能优化
3. 批量操作增强
4. 错误处理优化
5. 空状态引导

## 六、技术要点

### 6.1 成绩科目说明
- **主科（满分150）**：语文、数学、英语
- **理科（满分100）**：物理、化学、生物（有赋分）
- **文科（满分100）**：历史、政治、地理（有赋分）

### 6.2 分页参数
- `page`: 页码（从1开始）
- `page_size`: 每页记录数

### 6.3 权限控制
- 超级管理员：所有功能
- 学校分管管理员：仅管理所属学校数据

## 七、预期成果

1. 前端与后端API完全对齐，无数据加载问题
2. 所有统计图表正确展示后端返回的数据
3. 用户体验流畅，操作直观
4. 代码结构清晰，易于维护
