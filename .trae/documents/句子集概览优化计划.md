# 句子集概览功能优化计划

## 一、背景分析

### 当前实现问题
1. **性能问题**：当前通过 `getPaginatedSentences` 分页加载所有句子数据（每页100条），然后在前端计算统计数据
2. **效率低下**：数据量大时需要多次请求，加载时间长
3. **无缓存机制**：每次进入页面都需要重新加载全部数据
4. **无刷新功能**：用户无法手动刷新统计数据

### 后端新接口
- **接口地址**：`GET /sentence/admin/stats`
- **返回数据结构**：
```json
{
  "total_sentences": 1000,
  "enabled_sentences": 800,
  "disabled_sentences": 200,
  "categories": [
    {
      "id": "uuid",
      "name": "分类名称",
      "total": 100,
      "enabled": 80,
      "disabled": 20
    }
  ]
}
```

## 二、优化目标

1. 使用后端统计接口替代前端计算
2. 提升数据加载速度和页面响应性能
3. 添加数据缓存和刷新功能
4. 优化数据可视化展示
5. 完善错误处理和加载状态

### 4.4 缓存机制

```typescript
const loadCachedData = (): StatsData | null => {
  try {
    const cached = localStorage.getItem(CACHE_KEY)
    if (cached) {
      const data = JSON.parse(cached)
      if (Date.now() - data.lastUpdated < CACHE_DURATION) {
        return data
      }
    }
  } catch (e) {
    console.error('读取缓存失败:', e)
  }
  return null
}

const saveCachedData = (data: StatsData) => {
  try {
    localStorage.setItem(CACHE_KEY, JSON.stringify({
      ...data,
      lastUpdated: Date.now()
    }))
  } catch (e) {
    console.error('保存缓存失败:', e)
  }
}
```
