# 句子管理页面筛选功能优化计划

## 一、问题分析

### 1. "清除筛选"按钮显示问题

**当前代码位置**：`src/components/SentenceFilterCard.vue` 第104-109行

```vue
<div class="clear-filter-wrapper">
  <el-button v-if="hasActiveFilters" type="danger" @click="clearAllFilters" class="clear-filters-btn" size="large">
    <el-icon><Delete /></el-icon>
    清除筛选
  </el-button>
</div>
```

**问题**：

* `clear-filter-wrapper` 容器始终存在，即使按钮不显示

* 这导致布局中有一个空的占位空间

* 当按钮显示/隐藏时，布局会发生变化

### 2. 页面抖动问题

**原因分析**：

* `.filter-controls` 使用 `grid-template-columns: 1fr auto auto` 布局

* `filters-section` 中的元素宽度会随"清除筛选"按钮的显示/隐藏而变化

* `.active-filters-bar` 的显示/隐藏会导致整体高度变化

* 缺乏过渡动画，状态切换过于突兀

## 二、优化方案

### 优化1：改进"清除筛选"按钮容器

**修改**：移除始终存在的 `clear-filter-wrapper`，改为条件渲染整个容器

```vue
<!-- 优化前 -->
<div class="clear-filter-wrapper">
  <el-button v-if="hasActiveFilters" ...>
    清除筛选
  </el-button>
</div>

<!-- 优化后 -->
<Transition name="fade-slide">
  <el-button v-if="hasActiveFilters" ...>
    清除筛选
  </el-button>
</Transition>
```

### 优化2：固定布局结构，避免抖动

**方案**：

* 为 `filters-section` 设置固定最小宽度

* 使用 CSS 过渡动画平滑状态变化

* 为 `active-filters-bar` 添加进入/离开过渡

### 优化3：添加过渡动画

**新增CSS**：

```css
/* 按钮淡入淡出 */
.fade-slide-enter-active,
.fade-slide-leave-active {
  transition: all 0.3s ease;
}

.fade-slide-enter-from,
.fade-slide-leave-to {
  opacity: 0;
  transform: translateX(-10px);
}

/* 筛选条件栏过渡 */
.active-filters-bar {
  transition: all 0.3s ease;
}
```

## 五、预期效果

1. **"清除筛选"按钮**：仅在存在有效筛选条件时显示，无筛选条件时完全不占空间
2. **页面稳定性**：筛选操作时页面不再抖动，过渡平滑
3. **用户体验**：添加过渡动画，状态切换更加自然流畅
